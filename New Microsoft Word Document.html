1. Repository

A GitHub repository is a version-controlled folder.
Commands related to repository creation and setup:

Purpose Command
Initialize a local repo git init
Check repo status git status
Add a remote GitHub repo git remote add origin <repo_url>
View current remotes git remote -v
Remove a remote git remote remove origin
Rename a remote git remote rename origin <new_name>
Push local repo to GitHub git push -u origin main
Pull updates from remote git pull origin main
Fetch latest commits from remote (without merge) git fetch origin

Tip:
For first-time repo creation directly from CLI:

git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/<username>/<repo>.git
git push -u origin main

2. Commit

Commits represent snapshots of your project’s progress.

Purpose Command
Stage files git add <filename>
Stage all changes git add .
Unstage files git reset <filename>
Commit with message git commit -m "your message"
Amend last commit git commit --amend
Show commit history git log
Show one-line commit history git log --oneline
Show changes before committing git diff
Revert a specific commit git revert <commit_id>
Reset to previous commit (dangerous) git reset --hard <commit_id>

Tip:
Use git commit -am "message" when modifying tracked files (it skips the git add step).

3. Branch

Branches isolate development work.

Purpose Command
List branches git branch
Create a branch git branch <branch_name>
Switch to a branch git checkout <branch_name>
Create and switch in one go git checkout -b <branch_name>
Delete local branch git branch -d <branch_name>
Delete remote branch git push origin --delete <branch_name>
Merge branch into current branch git merge <branch_name>
View merged branches git branch --merged
View unmerged branches git branch --no-merged
Rename current branch git branch -m <new_name>

Tip:
To synchronize branch list after deleting a branch on GitHub:
git fetch -p

4. Pull Request

Pull Requests (PRs) are managed primarily on GitHub UI, but can be done via CLI using GitHub CLI (gh).

Purpose Command (GitHub CLI)
Authenticate GitHub CLI gh auth login
Create a pull request gh pr create --base main --head <branch_name> --title "PR Title" --body "Description"
View all pull requests gh pr list
View a single PR gh pr view <pr_number>
Checkout a PR locally gh pr checkout <pr_number>
Merge a PR gh pr merge <pr_number>
Close a PR without merging gh pr close <pr_number>

Tip:
Install GitHub CLI:
winget install --id GitHub.cli

5. Clone

Cloning creates a local copy of a GitHub repository.

Purpose Command
Clone repository git clone <repo_url>
Clone into specific folder git clone <repo_url> <folder_name>
Clone a specific branch git clone -b <branch_name> <repo_url>
Update local clone (fetch + merge) git pull

Example:
